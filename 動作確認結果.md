```
http://localhost:3000
```

### 2. 画像をアップロード
- 画像ファイルをドラッグ&ドロップ
- またはクリックしてファイルを選択

### 3. 確認ポイント
- [ ] 画像がアップロードされる
- [ ] ローディング表示が表示される
- [ ] 分析結果が表示される（カラーパレット、質感、トーン等）
- [ ] エラーメッセージが表示されない

### 4. エラーが発生した場合

#### ブラウザのコンソールを確認
1. F12キーを押して開発者ツールを開く
2. Consoleタブを開く
3. エラーメッセージを確認

#### よくあるエラーと対処法

**「APIキーが設定されていません」**
- `.env.local` ファイルを確認
- 開発サーバーを再起動

**「分析に失敗しました」**
- Gemini APIキーが有効か確認
- ネットワーク接続を確認
- ブラウザのコンソールで詳細なエラーを確認

**「モデルが見つかりません」**
- モデル名を確認（`gemini-1.5-flash` または `gemini-pro`）
- APIキーの権限を確認

## 🔧 トラブルシューティング

### 開発サーバーが起動しない
```bash
# ポート3000を解放
lsof -ti:3000 | xargs kill -9

# 再起動
cd ~/design_ai/app
npm run dev
```

### 環境変数が読み込まれない
```bash
# .env.localファイルを確認
cd ~/design_ai/app
cat .env.local

# 開発サーバーを再起動
npm run dev
```

### APIエラーが発生する
1. ブラウザのコンソールでエラーを確認
2. ネットワークタブでAPIリクエストを確認
3. エラーメッセージに基づいて対処
